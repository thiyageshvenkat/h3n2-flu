# H3N2 Flu Pipeline (WIP)
> A Nextflow-based computational pipeline for the structural and sequence analysis of H3N2 Influenza A sequences. This workflow integrates sequence parsing, homology modeling, structural stability evaluation, and protein language model (ESM-2) embeddings to generate structural graphs for downstream machine learning tasks.

![Alt](https://repobeats.axiom.co/api/embed/88738aac43feb67ab56145dd0a35ebd20b314e11.svg "Repobeats analytics image")

## 🧬 Pipeline Overview

The pipeline executes the following steps:

1. **FASTA Processing (`SPLIT_FASTA`)**: Splits multi-FASTA files (e.g., from GISAID) into individual sequences.
2. **Homology Modeling (`BUILD_HOMOLOGY_MODEL`)**: Generates 3D structural models of the sequences using **Modeller**, based on a template structure (default: `4O5I.pdb`).
3. **Structure Refinement (`CLEAN_PDB`, `REPAIR_PDB`)**: Cleans the generated PDB files and repairs structures using **FoldX**.
4. **Stability Analysis (`CALC_STABILITY`)**: Calculates the thermodynamic stability of the repaired structures using FoldX.
5. **Distance Metrics (`CALC_HAMMING`)**: Calculates the Hamming distance between the sequences and the template structure.
6. **Protein Language Modeling (`RUN_ESM2`)**: Generates embeddings and perplexity scores for each sequence using the **ESM-2** model.
7. **Graph Generation (`GENERATE_GRAPH`)**: Combines the 3D structural data (cleaned PDBs) and ESM-2 embeddings to construct graph representations (`.graph.pt`) suitable for Graph Neural Networks (GNNs).
8. **CPG Analysis** (`CALC_CPG`): Calculates CpG dinucleotide depletion ratios to evaluate how the virus evades human ZAP-mediated immunity.

## 🛠 Prerequisites

- [Nextflow](https://www.nextflow.io/docs/latest/getstarted.html) (21.04.0)
- Docker (The pipeline uses the `vulcan:latest` container image)
- [DVC](https://dvc.org/) (Data Version Control) for managing large datasets/models.

## 🚀 Usage

1. **Clone the repository:**
   ```bash
   git clone https://github.com/thiyageshvenkat/h3n2-flu.git
   cd h3n2-flu
   ```

2. **Prepare your data:**
   Place your GISAID multi-FASTA sequence file in the `data/` directory and ensure your template PDB is available.
   - Default sequence input: `data/gisaid.fasta`
   - Default template: `data/4O5I.pdb`

3. **Run the pipeline:**
   ```bash
   nextflow run main.nf -with-docker vulcan:latest
   ```

## 📂 Output Directory Structure

The pipeline publishes results in the `results/` directory:

- `results/split_sequences/`: Individual FASTA files.
- `results/models/`: Raw homology models generated by Modeller.
- `results/cleaned/`: Cleaned PDB structures.
- `results/repair/`: Repaired PDB structures from FoldX.
